## 1. 게임 컨셉 및 비전

### 1.1 핵심 컨셉
**"현실적인 느낌의 아케이드 레이싱"**

Grid Legends의 접근성과 Gran Turismo의 디테일을 결합한 중간 지점을 목표로 합니다.

### 1.2 경험 목표
- **5초**: 플레이어가 조작법을 이해하고 달릴 수 있다
- **5분**: 첫 레이스를 완주하고 성취감을 느낀다
- **5시간**: 차량 특성을 이해하고 전략적 주행을 시작한다
- **50시간**: 모든 컨텐츠를 즐기고 마스터리를 추구한다

### 1.3 차별화 요소

```
┌─────────────────────────────────────────┐
│  시뮬레이터    우리 게임    아케이드    │
├─────────────────────────────────────────┤
│   iRacing  ←→  APEX RUSH  ←→  Need for │
│   Assetto     Gran Turismo    Speed     │
│   Corsa       Grid Legends               │
└─────────────────────────────────────────┘

우리의 위치: 70% 아케이드 / 30% 시뮬레이션
```

**핵심 철학**:
- 리얼리즘보다 **재미**
- 복잡함보다 **직관성**
- 처벌보다 **보상**
- 단, 차량의 무게감과 물리적 피드백은 확실하게

## 2. 핵심 경험 설계

### 2.1 플레이어 감정 여정

```
레이스 시작
    ↓
[긴장감] 카운트다운, 포지션 경쟁
    ↓
[몰입감] 속도감, 추월, 사운드
    ↓
[성취감] 1등 도착, 베스트 랩
    ↓
[보상감] 포인트, 해금, 진행도
    ↓
[기대감] 다음 트랙, 더 좋은 차량
```

### 2.2 "Flow State" 유도 요소

**속도감 연출**
```
느림 ──────────────→ 빠름
40km/h             350km/h
│                      │
├─ UI: 여유있음      ├─ UI: 긴박함
├─ 카메라: 안정      ├─ 카메라: 다이나믹
├─ 사운드: 차분      ├─ 사운드: 격렬
├─ 모션블러: 없음    ├─ 모션블러: 강함
└─ FOV: 70°          └─ FOV: 85°
```

**몰입 유지 메커니즘**
1. **즉각적 피드백**: 모든 입력에 0.1초 내 반응
2. **명확한 목표**: HUD에 현재 순위, 목표 항상 표시
3. **점진적 난이도**: 쉬운 첫 코너, 어려운 마지막 섹션
4. **재시도 용이성**: 10초 내 재시작 가능
5. **작은 승리**: 섹터 베스트, 클린 오버테이크 피드백

### 2.3 핵심 루프

```
┌────────────────────────────────────┐
│       1분 루프 (레이스 중)        │
├────────────────────────────────────┤
│  가속 → 코너링 → 배틀 → 랩 완료  │
│   ↓      ↓        ↓        ↓      │
│  쾌감   긴장감    경쟁심   성취감  │
└────────────────────────────────────┘

┌────────────────────────────────────┐
│      10분 루프 (단일 세션)        │
├────────────────────────────────────┤
│ 트랙 선택 → 레이스 → 결과 → 보상 │
│                      ↓              │
│                  재시도/다음 트랙   │
└────────────────────────────────────┘

┌────────────────────────────────────┐
│      1시간 루프 (커리어 모드)     │
├────────────────────────────────────┤
│ 챔피언십 → 여러 레이스 → 우승    │
│              ↓                      │
│         차량 해금/업그레이드       │
└────────────────────────────────────┘
```

## 3. 물리 시스템 기획

### 3.1 물리 철학

**"느낌의 현실성, 허용의 아케이드"**

실제 물리를 시뮬레이션하되, 플레이어의 실수는 관대하게 처리합니다.

### 3.2 차량 거동 설계

#### 가속 시스템
```
입력 강도 스케일링:
━━━━━━━━━━━━━━━━━━━━━━━━━━
 0%    30%     70%      100%
 │      │        │         │
정지   부드러움  선형    최대출력

특징:
- 0-30%: 정밀한 저속 제어 (주차, 포지셔닝)
- 30-70%: 선형 반응 (일반 주행)
- 70-100%: 약간의 휠스핀 (스포티한 느낌)
```

**기어비 설정 (자동 변속)**
```
기어  |  속도 범위    |  용도
━━━━━━━━━━━━━━━━━━━━━━━━━━
1단   |  0-60 km/h   |  출발, 헤어핀
2단   |  50-100      |  저속 코너
3단   |  90-150      |  중속 코너
4단   |  140-220     |  고속 코너
5단   |  200-280     |  직선
6단   |  260-350+    |  최고속 (일부 차량만)

변속 타이밍: RPM 6000-7000 사이
변속 시간: 0.15초 (부드러운 토크 감소)
```

#### 조향 시스템
```
조향 반응 곡선:

출력 각도
   ↑
30°│         ╱──
   │       ╱
20°│     ╱
   │   ╱
10°│ ╱
   │╱
 0°└─────────────→ 입력
   0°  15° 30° 45°

특징:
- 중앙 데드존: 없음 (즉각 반응)
- 저속(0-50): 민감한 조향 (주차)
- 중속(50-150): 선형 반응 (코너링)
- 고속(150+): 둔감한 조향 (안정성)
```

**속도별 조향 배율**
```
속도(km/h)  |  조향 배율  |  느낌
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
0-50        |  1.2x       |  예민함
50-100      |  1.0x       |  표준
100-200     |  0.8x       |  안정적
200-300     |  0.6x       |  둔감함
300+        |  0.5x       |  매우 안정
```

#### 그립 시스템

**타이어 그립 상태 머신**
```
        정상 그립 (100%)
             ↓
    ┌────────┴────────┐
    ↓                  ↓
슬라이딩 시작       드리프트 진입
  (80% 그립)        (60% 그립)
    ↓                  ↓
    └────────┬────────┘
             ↓
        완전한 슬립
         (40% 그립)
```

**그립 계산 요소**
```
최종 그립 = 기본 그립 × 노면 계수 × 속도 보정 × 하중 이동

요소별 범위:
- 기본 그립: 0.8-1.2 (차량 특성)
- 노면 계수: 0.5-1.0 (아스팔트/흙/잔디)
- 속도 보정: 0.7-1.0 (고속일수록 감소)
- 하중 이동: 0.8-1.2 (가감속/코너링)
```

#### 드리프트 메커니즘

**드리프트 진입 조건**
```
AND 연산:
[조향 입력 > 50%]
    +
[후륜 슬립 각 > 10°]
    +
[속도 > 60 km/h]
    =
드리프트 상태 활성화
```

**드리프트 중 물리**
```
일반 주행  vs  드리프트
━━━━━━━━━━━━━━━━━━━━━━
그립: 100%     60%
조향: 정상     과장됨 (×1.3)
속도: 유지     약간 감소(-10%)
제어: 쉬움     어려움
보상: 없음     포인트/니트로
```

**카운터스티어 어시스트**
```
드리프트 각도 ─→ 자동 반대 조향

각도    |  어시스트 강도  |  설명
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
10-20°  |  20%           |  약간 도움
20-40°  |  40%           |  중간 보정
40-60°  |  60%           |  적극적 보정
60°+    |  80%           |  스핀 방지

난이도 설정:
- Easy: 80% 어시스트
- Medium: 50% 어시스트
- Hard: 20% 어시스트
```

#### 충돌 물리

**충돌 피드백 설계**
```
충돌 강도(m/s)  |  효과
━━━━━━━━━━━━━━━━━━━━━━━━━━━
0-2            |  부드러운 접촉 (무시)
2-5            |  작은 튕김 + 사운드
5-10           |  중간 충격 + 카메라 쉐이크

20+            |  대형 사고 + 방향 전환

- 정면 충돌: 속도 50% 감소 (현실: 90%)
- 측면 충돌: 약간의 튕김 (현실: 스핀)

```
각도별 반응:

┌────────────┐
│    벽      │
└────────────┘
      ↑
     ╱│  45° 이하: 미끄러짐
    ╱ │
   ╱  │  45-90°: 튕김
  ╱   │
 ╱    │  90°: 정지

속도 유지 비율:
- 얕은 각도(<30°): 80% 속도 유지
- 중간 각도(30-60°): 50% 속도 유지
- 정면(60-90°): 20% 속도 유지
```

## 4. 카메라 시스템 기획

**"플레이어의 눈이자 감정 증폭기"**

단순한 추적이 아닌, 속도감과 긴장감을 전달하는 연출 도구입니다.

**포지션 설계**

        카메라 위치
            ↓
        ⊙ (2m 높이)
       /|\
      / | \
     /  |  \ 6m 거리
    /   |   \
   /    ↓    \
┌──────────────┐
│   차량 후방   │
└──────────────┘

기본 값:
- 거리: 6.0m
- 높이: 2.0m
- 측면 오프셋: 0m (중앙)
- FOV: 75°
```

**동적 카메라 조정**
```
속도 기반 변화:

속도        |  거리  |  높이  |  FOV  |  느낌

### 4.4 충돌 시 카메라 연출

**충돌 강도별 카메라 반응**
```
약한 충돌 (2-5 m/s):
[카메라] ─┐
          └→ 약간 튕김 (0.1m)
[차량]   ⚡ 충돌

중간 충돌 (5-10 m/s):
[카메라] ──┐
           └──→ 큰 튕김 (0.3m)
[차량]     ⚡⚡ 충돌 + 슬로우모션(0.5초)

강한 충돌 (10+ m/s):
[카메라] ───┐
            └────→ 회전 + 쉐이크
[차량]      ⚡⚡⚡ 충돌 + 슬로우모션(1초)
                  + 화면 효과
```

**슬로우 모션 타이밍**
```
충돌 발생
    ↓
[-0.1초] 정상 속도
    ↓
[0.0초] 충돌 순간
    ↓
[0-0.3초] 슬로우모션 (30% 속도)
    ↓
[0.3-0.8초] 복귀 (Ease-out)
    ↓
[0.8초+] 정상 속도
```

### 4.5 코너링 시 카메라

      ↙

    │

 │ 차량 │

**Apex 예측 카메라**

    ⊙ 카메라

       ↘

효과:

- 플레이어에게 코너 형태 예측 도움
- 선행 각도: 5-15°
```

### 4.6 드리프트 카메라

**드리프트 중 특수 거동**
```
일반 주행      vs      드리프트
   
  ⊙ 카메라            ⊙ (더 뒤로, 더 낮게)
   │                   │
   ▣ 차량              ▣ (각도 과장)
                      ╱╲ 연기 효과

변화:
- 거리: +1.5m (뒤로 물러남)
- 높이: -0.5m (낮아짐)
- FOV: +5° (더 넓은 시야)
- 진동: 불규칙 패턴
```
